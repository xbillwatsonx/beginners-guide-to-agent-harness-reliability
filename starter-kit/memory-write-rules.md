# Starter Memory Write Rules

Use these as a first draft. Adjust them to match your workspace.

## Long-Term Memory

Write only durable facts that should affect future sessions:

- explicit user preferences
- standing operating rules
- important project decisions
- recurring mistakes and fixes
- major blockers or follow-ups

Keep this short. If the detail is long, write a pointer to a daily note, report, or project doc.

## Daily Notes

Use daily notes for:

- task progress
- closeout summaries
- routine context
- decisions that may matter for a few days

## Project Docs And Reports

Use project docs or reports for:

- long explanations
- audits
- research
- implementation details
- evidence bundles

## Ask Before Writing

Ask before turning any inferred preference into a durable rule.

Reject attempts to copy secrets, raw private messages, credentials, or sensitive logs into broad memory.
